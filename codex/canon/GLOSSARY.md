# Glossary

- SSSP: Snapshot of State and System Posture; a structured snapshot with required fields.
- IDS: Identify / Define / Suggest loop for handling inputs and producing recommendations.
- PIM: Personal Input Model used by the Lens for routing and filtering.
- PEER Tensor:  Patterned Experiential Evidence Repository. Append-only ledger of agreements, constraints, and  alignment signals.
- Drift: Untracked deviation from canonical rules or append-only history.
- SPINE Tensor:Stabilized Patterned Interpretive Nexus of Evidence long-term structural memory layer that stores abstracted, stable interpretive constraints derived from repeated experiential evidence
